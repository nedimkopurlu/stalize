## Gaps Found and Fixed During Audit

### Gap 1 — DATA-01: kap_parser still used BIST100_SYMBOLS

**Found by:** Integration checker at `kap_parser.py:302`  
**Impact:** KAP announcements for ~299 non-BIST100 stocks were parsed but silently dropped in `store_announcements` (no matching Stock record)  
**Fix:** `settings.BIST100_SYMBOLS` → `settings.BIST_FULL_SYMBOLS` at line 302  
**Verified:** `grep -n "BIST_FULL_SYMBOLS" kap_parser.py` returns line 302 ✓

### Gap 2 — LOGIC-04 frontend: PortfolioPosition.partial missing

**Found by:** Integration checker at `api.ts:398-410` and `portfolio/page.tsx`  
**Impact:** Backend emits `"partial": true` for positions without live prices; frontend interface dropped the field silently; users saw no indicator when price was unavailable  
**Fix 1:** Added `partial: boolean` to `PortfolioPosition` interface in `api.ts`  
**Fix 2:** Rendered amber `~` indicator in portfolio table when `pos.partial === true`  
**Verified:** Field in interface at line 409; render at `portfolio/page.tsx:197` ✓
