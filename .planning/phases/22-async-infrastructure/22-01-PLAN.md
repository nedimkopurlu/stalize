<objective>
Fix two classes of async safety issues in the backend: (1) blocking `time.sleep()` calls inside async service functions, and (2) manual `async with AsyncSessionLocal()` session management inside API route handlers that bypasses FastAPI's dependency-injected lifecycle.

Purpose: These are the two highest-impact reliability issues. `time.sleep()` inside async functions stalls the entire event loop for up to 30 seconds (3 retries × 10s) during yfinance rate limiting, blocking all concurrent requests. Manual session creation in routes means no guaranteed commit/rollback on exception and connection pool growth under concurrent load.

Output: `data_collector.py` and `macro_news.py` with all `time.sleep()` replaced by `await asyncio.sleep()` (or moved to executor-only scope where appropriate); `stocks.py`, `portfolio_v2.py`, `macro.py`, `admin.py` with every route handler receiving `db: AsyncSession = Depends(get_db)` as a parameter instead of creating sessions manually.
</objective>

async def get_db() -> AsyncSession:
    """Dependency for FastAPI route handlers."""
    async with AsyncSessionLocal() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()

       limit: int = Query(100, ge=1, le=200),
