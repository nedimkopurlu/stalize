<objective>
Fix two reliability issues in `backend/app/main.py`: (1) all 13 five-minute interval scheduler jobs currently have no `start_date`, causing a thundering herd at startup where every job fires simultaneously the moment the scheduler starts; (2) the two `asyncio.create_task()` calls at startup have no error tracking — if a task raises, the exception is swallowed silently and the application starts in a broken state with no indication.

Purpose: Thundering herd at startup saturates the database connection pool and yfinance rate limits simultaneously, causing cascading failures in the first minutes after every deploy. Silent task failures mean corrupted startup state is invisible until symptoms appear elsewhere.

Output: `main.py` with staggered `start_date` offsets on all interval jobs (30-second gaps between jobs) and a `_log_task_error` done-callback applied to every `create_task()` call.
</objective>

**Step 1 — Add `_log_task_error` helper function.**

Add this function near the top of `main.py`, BEFORE the `lifespan` function definition (after the scheduler global variable declaration, around line ~26):

```python
def _log_task_error(task: asyncio.Task) -> None:
    """Done-callback for startup asyncio tasks. Logs exceptions so failures are visible."""
    if task.cancelled():
        logging.warning("STARTUP_TASK_CANCELLED [%s]", task.get_name())
        return
    exc = task.exception()
    if exc is not None:
        logging.error(
            "STARTUP_TASK_ERROR [%s]: %s",
            task.get_name(),
            exc,
            exc_info=exc,
        )
```

**Step 2 — Wrap both `asyncio.create_task()` calls.**

Current code (lines ~283-284 inside `lifespan`):
```python
asyncio.create_task(startup_refresh_sources())
asyncio.create_task(data_collector.full_initial_load())
```

Replace with:
```python
_task_startup_refresh = asyncio.create_task(
    startup_refresh_sources(), name="startup_refresh_sources"
)
_task_startup_refresh.add_done_callback(_log_task_error)

_task_initial_load = asyncio.create_task(
    data_collector.full_initial_load(), name="full_initial_load"
)
_task_initial_load.add_done_callback(_log_task_error)
```

Note: The `name=` parameter is supported in Python 3.7+ and provides the task name visible in logs. The project is on Python 3.9, so this is safe.

**Result:** If either task raises an unhandled exception, `_log_task_error` fires and logs a line containing `STARTUP_TASK_ERROR` with the task name and full exception including traceback. The application continues running (does not re-raise), but the error is now searchable in logs.
