<objective>
Close TECH-01: every APScheduler job in `backend/app/main.py` gains `max_instances=1` and `misfire_grace_time=300` so that two back-to-back fires of the same cron/interval never run concurrently.

Purpose: The audit showed scheduler jobs could overlap themselves (e.g. a long yfinance fetch still running when the next 5-minute KAP scan fires). Overlap corrupts diskcache writes and doubles DB load. max_instances=1 forces the scheduler to skip or defer the second fire; misfire_grace_time=300 allows a 5-minute grace window before a missed fire is abandoned.
Output: Updated main.py with 10 hardened add_job() calls; pytest file proving the invariant by inspecting scheduler.get_jobs().
</objective>
