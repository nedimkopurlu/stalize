<objective>
Close TECH-02: wrap every synchronous yfinance call in `data_collector.py` and `macro.py` with `asyncio.get_event_loop().run_in_executor(None, ...)` so the FastAPI event loop is never blocked while Yahoo Finance HTTP calls are in flight.

Purpose: Audit found that `yf.Ticker(...).history()` and `yf.download()` are synchronous network calls made from inside `async def` handlers. A 3-second Yahoo latency freezes the entire API process. Wrapping with `run_in_executor(None, ...)` offloads the work to the default threadpool while keeping the caller's coroutine non-blocking.
Output: `data_collector.py` wrappers become async, internal callers updated to await them, `macro.py` indicators endpoint offloads yf.download, tests prove no regression and wrappers are coroutines.
</objective>
