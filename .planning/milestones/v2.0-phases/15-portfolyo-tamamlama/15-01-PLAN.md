Locate the GET /api/portfolio/history endpoint in backend/app/api/portfolio.py (currently returns snapshot rows from PortfolioDailySnapshot).

Make these changes:

1. After fetching snapshot rows (already sorted by date asc), compute cumulative portfolio return:
   - Base value = total_value of first snapshot (or 1.0 if first daily_pnl_pct is baseline)
   - For each row: portfolio_return_pct = ((total_value / first_total_value) - 1) * 100
   - If snapshots list is empty, return {"history": []} immediately

2. Fetch XU100.IS daily close prices from yfinance using run_in_executor (never call yf.download() directly in async context — TECH-02 pattern already used in macro.py):
   ```python
   import asyncio
   import yfinance as yf
   loop = asyncio.get_event_loop()
   def _fetch_xu100(start_date, end_date):
       df = yf.download("XU100.IS", start=start_date, end=end_date, progress=False)
       return df["Close"] if not df.empty else None
   xu100_series = await loop.run_in_executor(None, _fetch_xu100, start_date, end_date)
   ```
   Where start_date = first snapshot date (minus 1 calendar day as buffer), end_date = last snapshot date + 1 day.

3. Calculate BIST100 cumulative return:
   - xu100_base = first close price on or after the first snapshot date
   - For each snapshot date: look up xu100_series by date index; bist100_return = ((price / xu100_base) - 1) * 100
   - If a date has no XU100.IS data (weekend gap in series), use None for that day

4. Merge into response: each history item gains `portfolio_return` (float) and `bist100_return` (float | null)

Do NOT use mock or fallback price data. If XU100.IS fetch produces an empty DataFrame, set all bist100_return values to null and log a warning.
